### 具体步骤

#### 1. 检查基础环境

在 PowerShell 或终端中依次输入以下命令：

```bash
git --version
node -v
npm -v
openclaw --version
python --version
```

如果这些命令都能正常显示版本号，说明新电脑的基础工具基本可用。
如果某个命令报错，先记录报错内容，再判断是软件没有安装、环境变量没有配置，还是权限问题。

#### 2. 检查 OpenClaw 和钉钉插件

确认 OpenClaw 是否能正常启动，并检查钉钉插件是否安装成功。
重点查看以下内容：

1. OpenClaw 是否能正常运行；
2. 钉钉插件是否已经安装；
3. 钉钉账号是否已经连接；
4. 机器人是否绑定到正确的钉钉账号或群聊；
5. 运行过程中有没有报错信息。

如果之前创建的机器人找不到，可以重新创建一个新的钉钉机器人，但要保证记录清楚新机器人的配置步骤。

#### 3. 测试机器人最小回复

今天最核心的任务是测试这条链路：

用户发送“你好” → 钉钉机器人接收消息 → OpenClaw / 后端处理 → 机器人回复“收到”

测试时先不要加入大模型 API，也不要加入复杂功能。
只要机器人能固定回复“收到”，就说明消息收发链路已经基本打通。

#### 4. 如果机器人没有回复，按顺序排查

如果发送“你好”后仍然没有回复，可以按照下面顺序排查：

1. 确认 OpenClaw 是否正在运行；
2. 确认钉钉账号是否连接成功；
3. 确认机器人是否添加到了正确的会话；
4. 确认插件是否正常启动；
5. 查看终端是否有报错；
6. 检查是否存在权限、网络或配置问题；
7. 重新创建机器人并重新绑定测试。

排查时每发现一个问题，都记录“现象、可能原因、解决尝试、结果”，方便之后写实习笔记和项目文档。
