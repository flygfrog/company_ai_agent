# 技术方案

## 总体架构

用户在钉钉发送消息 → 钉钉机器人接收消息 → Python 后端服务接收 → AI Agent 调度模块 → 构造 prompt 和 messages → 调用 DeepSeek API
→ DeepSeek 返回回答 → AI Agent 整合结果 → Python 后端服务封装回复 → 钉钉机器人返回消息给用户

## 技术选型

- 编程语言：Python
- 后端框架：FastAPI
- 服务运行：Uvicorn
- 消息入口：钉钉机器人
- 大模型调用：DeepSeek API
- Agent 调度：自定义轻量调度模块
- 部署方式：Docker
- 配置管理：.env 环境变量
- 日志记录：Python logging
- 安全校验：钉钉签名校验
- 后续扩展：知识库、数据库、Nginx、LangChain / LangGraph
